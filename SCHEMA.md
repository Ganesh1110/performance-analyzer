# Performance Report Schema

This document describes the structure of `performance_report.json` generated by the React Native Performance Analyzer.

## Root Properties

All keys present at the root of `performance_report.json`:

* **`timestamp`** (string): The ISO 8601 timestamp of when the analysis was run.
* **`summary`** (object): General overview metrics.
  * `totalFrames` (number): Total frame samples parsed.
  * `bottleneckCount` (number): Number of detected frame drops.
  * `reRenderIssueCount` (number): Number of components flagged with re-render issues.
  * `hierarchyIssueCount` (number): Number of parent-child cascading re-renders.
  * `contextCascadeCount` (number): Number of context-driven cascading re-renders.
  * `memoryLeakCount` (number): Number of potential memory leaks.
  * `jsBottleneckCount` (number): Number of JS thread bottlenecks.
  * `navigationIssueCount` (number): Number of screen transitions exceeding budgets.
  * `flatListIssueCount` (number): Number of list components with rendering issues.
  * `healthScore` (number): A score from 0-100 indicating the general performance health.
* **`componentPaths`** (object): Map of component name strings to local source file absolute paths.
* **`flashlightStats`** (object): Summary statistics for native metrics (fps, cpu).
  * `fps` (object): min, max, avg, median, p95, p99.
  * `cpu` (object): min, max, avg, median, p95, p99.
* **`bottlenecks`** (array of objects): Frame drop events correlated with React commits and network activity.
* **`reRenderIssues`** (array of objects): Detailed component-level re-render statistics.
* **`memoryAnalysis`** (object): RAM timeline, memory trends, spikes, leaks, and component correlations.
* **`hierarchyIssues`** (array of objects): Parent-to-child re-render cascade issues.
* **`contextCascades`** (array of objects): Multi-component context re-render cascade events.
* **`bundleAnalysis`** (object, optional): Metrobundle file size statistics, large component warnings, and size-to-render-time correlation.
* **`flows`** (array of objects): User performance flow metrics compared to budgets.
* **`anomalies`** (array of objects): Statistical anomalies in native metrics.
* **`jsBottlenecks`** (array of objects): JS thread bottleneck spikes grouped consecutively.
* **`navigationAnalysis`** (array of objects): Screen transition durations and FPS within transition windows.
* **`flatListAnalysis`** (array of objects): FlatList rendering efficiency metrics.
* **`animations`** (array of objects): Screen animations and their smoothness metrics.
* **`concurrentAnalysis`** (object): Concurrent React features coverage (transitions, suspenseBoundaries, deferredUpdates, interruptedRenders).
* **`phaseAnalysis`** (array of objects): Breakdown of render/commit phase durations for React commits.
* **`prediction`** (array of objects): AI-based performance prediction suggestions for components.
* **`executiveSummary`** (string): Natural language summary generated by the NL reporter.
* **`automatedFixes`** (array of objects): Automated code changes recommended for re-render issues.
* **`networkSummary`** (object, optional): General summary of network activity logs.
